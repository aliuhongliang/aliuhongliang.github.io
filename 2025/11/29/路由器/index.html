路由器配置

  • 方式1: 让路由器可以使用ssh，进入luci页面，进入网络 - 防火墙 - 通讯规则 - 开启allow-ssh 即可
  • 方式2: 让路由器可以使用ssh，网线接入电脑，直接进入 ssh root@192.168.1.1

dnsmasq配置

openwrt 上的 dnsmasq 配置中 不支持 –add-subnet,需要改造启动文件
进入 /etc/init.d/dnsmasq 找到添加配置的地方， 添加新的配置： append_parm “$cfg” “add_subnet” “–add-subnet”
进入 /etc/config/dhcp 修改 dnsmasq 配置 添加 option add_subnet ‘32,128’ list listen_address ‘127.0.0.1’ list listen_address ‘192.168.1.1’
重启： /etc/init.d/dnsmasq restart

linux命令部分：

  • 查看进程: ps

  • 查看 xxx 的进程：ps | grep https-dns-proxy，这里的 xxx 就是 https-dns-proxy

  • 查看文件： cat /etc/config/dhcp

  • 切换至root权限：sudo -i

  • 杀死 xxxx 进程：killall -9 xxxx

  • 复制文件到服务器：scp -O ~/Desktop/libcares1.34.5-1aarch64_cortex-a53.ipk root@192.168.10.150:/tmp/

    -O 是强制使用sftp，在路由器上的时候，不使用 -O copy总是失败
    ~/Desktop/libcares1.34.5-1aarch64_cortex-a53.ipk 要拷贝的文件
    root@192.168.10.150:/tmp/ copy到服务器下的tmp文件夹

  • 复制服务器文件到本地： scp -r ubuntu@192.168.10.212:/home/ubuntu/openwrt/MT7981-MT3000/build_dir/target-aarch64_cortex-a53_musl/dnsmasq-nodhcpv6/dnsmasq-2.85/src ./

  • 复制服务器文件到本地：scp -O root@192.168.1.1:/etc/init.d/dnsmasq ./

  • 安装包命令：opkg install xxxx

  • 移除某个文件 rm /etc/config/doh_log.log

  • 文件移动，并重命名 mv /etc/config/doh_log.log /tmp/doh.log

  • udp捉包 tcpdump -i lo udp port 5053

  • udp捉包并写入文件 tcpdump -i lo udp port 5053 -w d0.pcap
