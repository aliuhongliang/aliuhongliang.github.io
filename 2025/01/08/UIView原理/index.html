UIView原理

发表于 2025-01-08 更新于 2025-09-05 分类于 iOS
本文主要介绍 UIView的显示原理、掉帧、性能优化。

UIView和CALayer的区别

  1. UIView 继承自 UIResponser，每个 UIView 都拥有一个 CALayer ，UIView为CALayer提供内容，主要负责事件处理，参与响应链
  2. CALayer 负责内容的显示 属性 contents 就是需要显示的内容是一个 bitmap 位图
  3. 应用的设计原则是：单一职责原则

UIView的显示原理

  • 总体流程

    1. UIView 的显示是CPU和GPU共同作用的结果，CPU负责UI布局，文本计算，绘制，图片的编解码，最后生成一个位图，通过Core Animate 提交给 GPU
    2. GPU对位图进行渲染、纹理合成等，最后放到帧缓冲区等待显示
    3. 视频控制器根据vsync同步信号，从帧缓冲区获取内容显示

  • UIView 的绘制原理

  1. 当调用 [UIView setNeedsDisplay] 时，会直接调用 layer的同名方法 [view.layer setNeedsDisplay] 来标记需要绘制

  2. 等待 runloop 当次循环将要结束的时候，触发 runloop 的 observer 回调后，调用 [CALayer display] 进行绘制

  3. 判断 layer 的 delegate 是否实现了代理方法 displayLayer:

    3.1 实现了该代理进入异步绘制

    UIView通过实现CALayer的代理方法来进入异步绘制，代码如下： - 自定义 UIView 的异步绘制 - 自定义 CALayer 的异步绘制

    3.2 未实现该代理进入系统绘制

    - 创建一个context(每个layer都会有一个context,这个context指向一块缓存区) - layer 有代理，调用 - (void)drawLayer:(CALayer *)layer inContext:(CGContextRef)ctx 方法, 进行绘制, 之后通过调用 - (void)drawRect:(CGRect)rect; 给开发者提供一个可以在系统绘制基础的绘制函数 - layer 没有代理，调用 -[CALayer drawInContext:] 方法 >1. 系统的UIView的layer的代理就是自己，所以系统创建的 UIView 的系统绘制会执行 - (void)drawLayer:(CALayer *)layer inContext:(CGContextRef)ctx 方法，之后调用 - (void)drawRect:(CGRect)rect; >2. drawInContext: 方法当我们使用CALayer或者子类的时候，没有设置代理，可以重写该方法，来实现绘制

  4. 绘制完成后，通过CALayer提交位图到GPU渲染

  • 总结

    UIView 的绘制工作主要是CPU 完成的，主要任务是 UI布局包括文本计算，frame设置，绘制，图片的编解码，提交位图到GPU

  • GPU 渲染原理
    1. 顶点着色 / 形状(图元)装配 / 几何着色 / 光栅化 / 片段着色 / 提交到帧缓冲区
    2. 图形渲染使用的技术栈 Core Graphics、Core Animation、Core Image、OpenGL、metal

UI卡顿、掉帧

以刷新帧率60fps为例，大概是16.7ms就需要有一帧画面需要显示，如果超过这个时间，人眼就会有卡顿感觉。
主要是两方面

  1. cpu：cpu耗时太多，留给gpu工作时间太短，导致在16.7ms内不能完成一张图片的渲染
  2. gpu：gpu耗时太多在16.7ms内未能完成一张图片的渲染

离屏渲染

  1. 在屏渲染
    GPU的渲染工作是在当前屏幕缓冲区内进行的
  2. 离屏渲染
    GPU的渲染工作，需要在当前屏幕以外重新开辟缓冲区进行，就是离屏渲染。
    如何理解离屏渲染？
  1. 什么是离屏渲染
    某些复杂的图形效果无法直接在当前屏幕区域进行绘制，需要通过离屏缓冲区来绘制。离屏渲染是指GPU在当前屏幕缓冲区外开辟了一个新缓冲区来进行渲染工作
  2. 什么时候会触发离屏渲染
    • cornerRadius和maskToBounds组合使用
    • 设置蒙版mask
    • 阴影
    • 光栅化
  3. 为什么要避免离屏渲染
    • 性能问题：触发离屏渲染会增加GPU的工作耗时，进而导致总时间超过视频同步信号的频率时间，会造成UI卡顿现象
    • 内存占用：离屏渲染会开辟新的内存空间，作为缓冲区
    • GPU负载：离屏渲染要把所有的渲染结果合成后一起显示，多了很多额外的计算以及上下文的切换。都会对GPU造成额外的开销

光栅化

光栅化: 是一种优化图层渲染性能的技术，尤其适用于有复杂效果（如阴影、滤镜、模糊等）的图层。通过将图层的复杂效果预先计算为位图，可以避免每一帧重新计算这些效果，从而减少计算量、内存带宽的压力，并提高 GPU 渲染效率

性能优化 UITableView

  1. CPU优化
    • 轻量级对象的使用，比如CALayer代替UIView
    • 复杂计算放入子线程，比如cell高度计算、图片的编解码
    • 避免动态创建销毁视图对象
    • 异步绘制，减少视图层级
  2. GPU优化
    • 避免触发离屏渲染
    • 减少视图的层级结构，进而减少GPU的耗时
