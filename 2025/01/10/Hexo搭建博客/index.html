发表于 2025-01-10 更新于 2025-12-24 分类于 其他
如何使用Hexo搭建自己的博客

环境依赖

  • 依赖环境 node npm git，请自行安装

安装hexo

  • 安装hexo npm install -g hexo-cli
  • 创建本地文件夹 hexo_blog
  • cd hexo_blog
  • hexo init // 生成模板
  • npm install
  • hexo server 本地运行 （hexo s）

安装主题

  • git clone https://github.com/theme-next/hexo-theme-next themes/next
  • 修改hexo（根目录下）的 _config.yml 文件：theme: next
  • hexo g #或者 hexo generate
  • hexo s

    修改主题自己另行查阅 next 文档

写博客

  • hexo new page categories 添加分类

hexo new post “博客title” 添加博客正文

创建GitHub仓库

  • 新建一个仓库，名字必须是 GitHub用户名.github.io。
  • 修改根目录下的 _config.yml
  • npm install hexo-deployer-git –save
  • hexo clean
  • hexo g
  • hexo d

访问自己的博客

  • http://GitHub用户名.github.io

自动部署

  • 在github上新建一个 hexo_blog_source 的仓库，用来保存我们的博客源码
  • 本地新建一个文件夹，hexo_blog_source，将 hexo_blog 中的 .github，scaffolds，source，themes，_config.yml，package.json，package-lock.json，拷贝到 hexo_blog_source 中
  • 重点：将你themes中的 .git 删除
  • 按照 hexo_blog_source 仓库的要求，使用git，将代码push上去
  • 在github的个人中心，setting –> developer setting –> personal access token –> Tokens ,点击 Generate new token。
  • 勾选 repo 和 workflows，生成token，复制并保存。
  • 将token 应用到 hexo_blog_source 仓库中, 记住你定义的3个变量名,查看自己的node版本，在下面的代码中修改成自己的。





  • hexo_deploy.yml 文件
  • 编辑完成后，提交后就会触发工作流，将代码 博客静态文件 push 博客的远程仓库。
