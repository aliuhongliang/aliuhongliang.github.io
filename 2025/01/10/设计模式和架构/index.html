MVVVM

MVVM (Model-View-ViewModel) 将视图逻辑和业务逻辑解耦，明确职责分工：

  • Model：处理数据和业务逻辑，与后端交互。
  • View：负责 UI 展示，不包含业务逻辑。
  • ViewModel：处理业务逻辑，绑定 Model 和 View。

优点

  • 提升代码可测试性。
  • ViewModel 独立于 View，便于复用。
  • 数据绑定机制（如 KVO 或 ReactiveCocoa）简化界面更新。

组件化

  • 业务隔离：不同功能模块之间互不依赖。
  • 高复用性：组件可以在多个项目中复用。
  • 易扩展：新增功能时，只需增加新组件。

模块间通信

  • 方法 1：通过协议进行通信
  • 方法 2：通过 URL Scheme 或路由机制
  • 方法 3：通过中间件或服务管理器 ModuleManager

如何重构

  • 模块优先：优先重构高频使用或问题严重的模块。
  • 新功能使用新架构：老功能保持现状，新功能使用 MVVM 和组件化。
  • 创建适配层（Adapter Layer）来兼容老代码
