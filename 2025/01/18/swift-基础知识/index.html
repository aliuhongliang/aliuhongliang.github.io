swift-基础知识

发表于 2025-01-18 更新于 2025-12-24 分类于 swift
本文主要介绍 swift的基础知识。

结构体

  1. 结构体默认生成 初始化器（构造方法）可能是多个初始化器 ：目的是保证所有的存储属性初始化后都要有初始值
  2. 如果自己定义了初始化器，那么系统就不会自动生成了。
  3. 方法要修改结构体的属性值，需要使用mutating来修饰

协议

  • 可以定义 方法 属性 下标的声明，可以被枚举 结构体 类 遵守，多个协议用逗号隔开
  • 协议方法中不能有默认参数
  • 协议定义的属性必须是var
  • 协议定义的类方法 类属性 类下标 必须是static 不能使用class
  • 默认需要实现所有方法，后续添加可选
  • 如果给结构体定义协议，并且该协议方法需要修改结构体属性的值，需要在协议方法中用mutating修饰。而不能说协议方法不用mutating，在实现的时候用mutating修饰。
  • 协议定义初始化器的时候，如果不是final类，必须加上required，这是因为子类默认继承协议，为了要保证子类也实现协议方法，在父类使用required来修饰该协议方法
  • 协议可以继承
  • 协议组合 func test(obj: protocol1 & protocol2 & protocol3) {},协议组合中最多只能有一个类类型class，可以有多个协议类型
  • 只有类才能使用的协议，协议需要遵守 class AnyObject
  • 可选协议 使用 @objc 修饰后，使用optional

枚举

  • 有关联值

一个int占4个字节，最多的有3个int，所以需要24个字节来存储关联值，还有一个字节用来存储成员值，就是是取的test1，还是test2这种，所以总共需要25个字节，有需要内存对齐，分配32个字节。

  • 无关联值

用一个字节用来存储成员值，就是是取 test1，还是test2这种

  • rawValue 获取枚举的原始值。【内部实现是使用的计算属性】
  • 可以定义类型属性/计算属性（本质是方法）
  • 由于可选是枚举类型，有两个值，一个是none 一个是some()可以用下面的写法

对象

  1. 没有生成可以传入属性值的初始化器，所以类的属性必须要告诉编译器它是有值的
  2. 如果类的属性都有值，编译器会生成无参的初始化器
  3. 内存，前8个字节存储类的相关信息(指向类对象的地址)，再8个字节存储引用计数信息，后面就是属性的存储字节数。

泛型

  • 类、结构体、枚举都可以使用泛型
  • 协议不能直接使用泛型，其采用 抽象类型成员（Abstract Type Member） 的方式，具体技术称为 关联类型（Associated Type）。

思考：

  • 协议为什么不能使用泛型？

    • 协议的设计，可以解决多继承，这也就说明协议可以被多个类遵守
    • 而swift是强类型语言，那么会导致类型的不确定性

防止继承的方式

  • final
  • static
  • private / fileprivate
  • 枚举
  • 协议与扩展: 防止具体实现被继承，通过协议提供功能

属性

  • 存储属性

    • 结构体、类有
    • 枚举没有
    • 创建类或者结构体的时候，所有的存储属性必须设置一个合适的初始值
    • 存储属性不允许被class修饰

  • 计算属性

    • 本质其实是一个方法，不占用实例对象的内存。
    • 结构体、类、枚举都可以有

  • 延迟属性

    • 表示该属性第一次被使用的时候才初始化
    • 如果多线程同时第一次访问lazy属性，无法保证属性被初始化1次 （非线程安全）
    • 不能使用 let
    • 结构体包含延迟属性的时候，创建该结构体如果使用let修饰，不能访问延迟属性，只能使用var修饰才可以访问延迟属性
      1. 结构体是值类型，使用let修饰，说明结构体的内存不能被修改，而延迟属性的特点是访问的时候才被赋值，赋值操作就是修改内存，所以使用let和存储属性相违背，会报错
      2. 结构体使用lazy可以用来优化内存

  • 属性观察器

    • 非lazy的var存储属性，才可以设置属性观察器
    • 初始化方法中设置属性值，不会触发willSet、didSet
    • 可以在子类中为父类增加属性观察器（除了只读，let属性）
    • 子类中初始化赋值会触发父类的属性观察器

  • 类型属性

    • 整个程序运行过程中，就只有一份内存(类似全局变量)，不占用所在类的内存
    • 不需要在初始化的时候给定值
    • 在第一次使用的时候被初始化
    • 线程安全的
    • 可以使用 let 因为：类型属性不存在初始化的问题。
    • static 修饰
    • 如果是类，使用static 或者 class来定义类型属性

static 和 class

  • static 【静态】
    • 修饰 存储属性 计算属性 类型方法
    • static修饰的类方法不能继承和重写；
    • 使用范围：class, struct, enum
  • class 【类】
    • 修饰 计算属性 类型方法
    • 存储属性 不能用 class 修饰，因为它属于实例级别，而 class 主要用于类型级别的属性
    • class修饰的类方法可以继承
    • 使用范围：class

      协议中使用class定义的方法，结构体不能使用该协议，因为class定义的方法，只能类来使用，结构体不行
      协议中使用static定义方法，实现的时候不能使用class
      协议中使用 class 定义方法，实现的时候能使用 static
      如果想要协议被子类继承，选择class 定义方法，因为static不能继承

Any AnyObject some

  • Any 代表 任意类型 （结构体、类、枚举、函数类型）
  • AnyObject 代表 任意类类型
  • some 不透明类型,可以用来解决协议中有关联类型的情况，用some修饰后，结果只能是确定的一种类型

is as as？as！

  • is 判断是否为某种类型
  • as 强制类型转换
  • as? 类型转换 返回可选类型
  • as! 类型转换 并强制解包

元类型

  • X.self Person.self: 元类型的地址
  • X.Type 是一个类型 X.self 的类型是 X.Type
  • AnyClass 的类型是 Anyobject.Type
  • 类.self 理解为获取的是类对象 ，类对象的类型是 类.Type. 任意类对象的类型 是Anyobject.Type 也就是AnyClass
  • type(of: 实例) 获取对象的类型 它的值是类对象 类型是 X.Type。【它的实质不是调用方法，而是直接将实例对象的前8个字节取出来返回】
  • 应用
  • self 表示当前对象。
  • Self 表示当前类型。一般作为返回值类型，可以类比oc中的instancetype，表示当前类型。Self可以直接使用 Self.count 访问count类属性
  • class_getInstanceSize() 获取该类型的内存大小
  • class_getSuperClass() 获取父类

String Array

  • String 底层

    • 占用内存 16个字节
      • 如果不超过15个字符，直接用16个字节存储字符串的ASCII码值。一个字节标识长度
      • 超过15个字节，前8个字节存储字符串长度， 后8个字节存储的是字符串真正内容的地址，存储在常量区
      • 字符串拼接后，如果未超过15个字符，依然存储在地址中。如果超过，需要新分配内存来存储，新的内存在堆区。在堆区malloc开辟新空间存储
    • 字符串的初始化是动态绑定的

  • Array 底层

    • 占用内存 8个字节
      • 存储的是堆空间的地址，这个地址指向的内存就是用来存储arr的信息。前32个字节存储arr信息，比如引用计数，数组长度，数组容量等，后面存储arr里的值

  • 动态数组是怎么实现的？一般是新开辟新的内存，然后将老的值copy到新内存

  • 字符串一般放在常量区

  • 可选类型的本质 是枚举

访问控制

  • fatalError assert 断言闪退，退出app
  • 访问控制
    • open ：模块中和模块外 可以访问和继承，只能类使用，结构体不行
    • public： 模块中和模块外可以访问，模块外不能继承
    • internal：模块中可以访问，模块外不能访问
    • fileprivate： 只能在当前文件中访问。
    • private:： 只能在定义的{ }中，和当前文件中 该类或结构体的扩展中访问
    • 类是fileprivate，里面的属性、方法默认都是fileprivate，如果要自定义只能定义比fileprivate低的访问权限
  • 访问权限
    • 禁止使用init private init() {}
